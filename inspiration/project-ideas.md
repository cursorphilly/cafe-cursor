# Project ideas for co-work

Copy-paste prompts for open build time. Pick one, or bring your own backlog.

---

## AR zine (featured)

Fork [zine-ar](https://github.com/luiscielak/zine-ar) and add **page 3**:

> I forked zine-ar (ar-zine-starter). Help me add a fourth zine page: create artwork in targets/page-3.png (high contrast, busy collage style), add an entry to src/content.js with one sticker layer and one 3D model, then walk me through npm run compile-targets.

---

## Grill Me your side project

> Grill me on my plan for [describe project] before we write any code. Ask about edge cases, auth, empty states, and what success looks like. Don't implement until I approve your summary.

---

## Approve the plan

> I want to build [feature]. Write a short plan with acceptance criteria. Grill me on it. After I approve, implement only what we agreed — flag any drift at the end.

---

## Ship something small

> I have 90 minutes. Help me ship the smallest useful version of [idea] — one screen, one API call, or one automation. No scope creep.

---

## Learn the codebase

> I'm new to this repo. Walk me through the architecture by asking me what I already know, then explain the gaps. Don't refactor anything.

---

## Personal software

> Help me scaffold a simple personal tool for [problem] — a single HTML page or markdown-driven dashboard I can iterate on weekly. Start with AGENTS.md describing my constraints.

Clone [business-os-starter](https://github.com/luiscielak/business-os-starter) and type `new-os` for a fuller version.

---

## Design → prototype

> I have a rough idea for [app]. Run /new-project style discovery: who it's for, core flow, then a clickable HTML prototype. I'm a designer, not a daily coder.

See [ai-product-design-rules](https://github.com/luiscielak/ai-product-design-rules).

---

## Debug with the room

> I'm stuck on [error or behavior]. Help me form a minimal repro and a one-paragraph explanation I can ask someone at the next table about.

---

## Explore limits

> Build something intentionally silly in 30 minutes — a useless Chrome extension, a joke API, or an AR sticker that makes no sense. Goal is learning what breaks, not shipping.
